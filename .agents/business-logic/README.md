# Business Logic

This folder defines the product meaning in plain language.

## Required Questions

Every business note, idea, and lesson definition should answer:

1. What does the user want to achieve?
2. What should animator software do to make that possible?
3. What should the user see, and does the UX meet the highest standard?
4. What should the user learn visually and step by step?

## Domains

- `content/` = lesson goals, user story, visible teaching path, and step-by-step learning
- `software/lesson-authoring/authoring/LESSON_AUTHORING.md` = the lesson writing contract
- `software/` = discovery, compilation, playback, and runtime behavior

## Rule

- Write the user-facing meaning before implementation detail.
