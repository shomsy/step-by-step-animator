# .agents

This folder holds the project rules and the project memory.

## Domains

| Domain | Responsibility |
|--------|----------------|
| `governance/` | Architecture, app structure, quality rules, review gates, and required questions |
| `business-logic/` | Business definitions for content and software behavior, anchored by `business-logic/README.md` |
| `management/` | Ideas, bugs, plans, TODOs, and history |
| `review/` | Review notes and review archive |

## Required Questions

Every idea, plan, and lesson decision should answer:

1. What does the user want to achieve?
2. What should animator software do to make that possible?
3. What should the user see, and does the UX meet the highest standard?
4. What should the user learn visually and step by step?

## Reading Order

1. `governance/`
2. `business-logic/`
3. `management/`
4. `review/`

## Rule

- Write the user language first, then the technical language.
- Keep the UX visible and the step-by-step learning path explicit.
- If the item cannot answer the required questions, it is not ready.
