# Naming Vocabulary

## Preferred Verbs

- `discover`, `read`, `validate`, `plan`, `build`, `create`, `find`, `write`, `publish`, `render`, `sync`

## Preferred Naming Feel

- banal
- intuitive
- predictive
- descriptive
- child-explainable

## Preferred Noun Direction

- use real flow or capability nouns such as `billing`, `checkout`,
  `invoice-delivery`, `route-resolution`, `runtime-handoff`, `user-import`
- prefer nouns that help the reader guess when and why the thing exists
- technical slices should still sound like owned capabilities, not junk drawers

## Avoid As Default

- `handle`, `process`, `manage`, `doWork`, `helper`, `util`, `service`, `manager`

## Rule

Folder says flow.
File says responsibility.
Function says exact action.

If the reader cannot predict what the folder, file, or function is for, the
name is not finished.
