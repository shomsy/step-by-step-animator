# New Contributor Flow

## Step 1: Read In Order

1. `.agents/README.md`
2. `AGENTS.md`
3. `.agents/governance/quality-gates.md`
4. `.agents/governance/profile-resolution-algorithm.md`
5. `.agents/governance/security/README.md`
6. `.agents/governance/app-architecture/README.md`
7. `.agents/governance/app-architecture/ARCHITECTURE.md`
8. `.agents/governance/how-to-code-review.md`
9. `.agents/governance/release-and-rollback-policy.md`
10. `.agents/business-logic/README.md`
11. `.agents/management/README.md`
12. `.agents/management/TIMELINE.md`

## Step 2: Understand Current State

- check active tasks in `management/TODO.md`
- check active bugs in `management/BUGS.md`
- check current board state in `management/ACTIVE.md`
- check latest evidence in `management/evidence/CHANGELOG.md`
- check release and test evidence when the task touches runtime or deployment

## Step 3: Deliver Safely

- follow coding and review standards
- update owning docs when boundaries change
- record evidence when finishing work
- use strict review when the work claims production readiness or major convergence
