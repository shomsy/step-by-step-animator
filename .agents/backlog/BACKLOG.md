# Backlog

This file captures raw ideas and user stories before they become planned work.
It is intentionally lightweight.

## Update Rules

- keep newest items at the top
- write the rough thought first; do not force decomposition
- use `YYYY-MM-DD HH:MM` timestamps so aging can be measured
- promote an item to `.agents/planning/PLAN.md` when it is selected for work
- move it to `.agents/evidence/TODO.md` only when execution is ready
- if the item is a bug or hotfix, promote it to `.agents/bugs/BUGS.md` instead of planning
