# AGENTS

This is the thin entry point for working in this repo.

- `.agents/README.md` is the project map.
- `.agents/governance/QUALITY.md` defines the required questions and quality rules.
- `.agents/governance/app-architecture/ARCHITECTURE.md` defines the repo architecture.
- `.agents/business-logic/README.md` defines the business meaning for content and software.
- `.agents/management/` holds backlog, bugs, planning, TODOs, and history.

When work changes, update the owning document under `.agents/`.
